# Docs Sync Rubric

Use para checar se documentacao acompanha mudancas de codigo, API ou release.

## Areas
- README e quickstart.
- Env vars e exemplos de config.
- API docs, schemas e exemplos de payload.
- Changelog e release notes.
- Runbooks, troubleshooting e rollback.
- Comentarios ou docs internos de arquitetura.

## Checklist
- Novos comandos estao documentados.
- Env vars novas nao incluem secrets reais.
- Contratos de API refletem comportamento real.
- Breaking changes tem migracao ou aviso.
- Exemplos foram atualizados.
- Docs obsoletas foram marcadas ou corrigidas.

## Saida recomendada
- Docs afetadas.
- Atualizacoes obrigatorias.
- Atualizacoes opcionais.
- Riscos se docs ficarem desatualizadas.
