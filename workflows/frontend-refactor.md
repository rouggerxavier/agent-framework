# Frontend Refactor Workflow

Use para refatorar UI preservando comportamento e qualidade visual.

## Sequencia
1. Use `repo-map-builder` para localizar telas, componentes e estado.
2. Use `ui-ux-pro-max-audit` para identificar problemas visuais e UX.
3. Use `implementation-planner` para dividir refatoracao.
4. Refatore com foco em componentes, estado e estilos existentes.
5. Use `test-strategy-builder` para cobertura de regressao.
6. Use `runtime-qa-audit` para validar fluxos e responsividade.
7. Use `diff-reviewer` antes de finalizar.

## Saidas
- Escopo de componentes.
- Plano de refatoracao.
- QA visual e funcional.
- Riscos de regressao.
- Checklist final.
